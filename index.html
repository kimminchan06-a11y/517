What changed

이번 구조 변경 포인트

  • 메인에서 로그인 폼을 제거하고 별도 로그인 페이지로 분리했습니다.
  • 상품 페이지는 쿠팡형 배열을 참고한 필터 + 정렬 + 카드형 목록 구조로 만들었습니다.
  • 결제창은 하나의 화면 안에서 결제 수단을 선택하도록 구성했습니다.
  • 선택 상품, 로그인 상태, 주문 조회 흐름이 서로 이어지도록 버튼 동작을 연결했습니다.
로그인 페이지로 이동하면 무엇을 할 수 있나요?

로그인, 회원가입, 주문 조회를 탭으로 전환할 수 있고, 결과 상태 카드를 통해 바로 상품 페이지나 주문 조회로 이동할 수 있습니다.

상품 페이지는 어떤 기준으로 벤치마킹했나요?

좌측 카테고리/브랜드/가격 필터, 상단 정렬 바, 촘촘한 카드 배열 구조를 참고하고 색감과 카드 비주얼은 현재 사이트의 유리 질감과 에메랄드 포인트에 맞춰 재해석했습니다.

결제 버튼을 누르면 실제로 무엇이 되나요?

현재는 MVP 단계라 실제 PG 연동 대신 주문 번호와 상태 카드가 생성되고, 로그인 페이지의 주문 조회 탭에서 같은 주문을 다시 확인할 수 있게 처리했습니다.
